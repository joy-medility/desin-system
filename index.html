PillEye Design System · v1.0

정확한 카운팅, 흔들림 없는 신뢰.

약사를 위한 비전 AI 기반의 PillEye B2B 환경에 맞는 다크모드 기반으로 신뢰감을 전달합니다. 절제된 색상 대비와 명확한 타이포그래피 위계, 즉각적인 상태 피드백을 통해 신뢰성을 화면 곳곳에서 시각적으로 구현해 더 나은 사용자 경험을 제공합니다.

Core Value

1. Trusted Accuracy
오류가 없는 곳에서, 신뢰가 시작됩니다.

Enterprise-grade counting accuracy with built-in error detection.

엔터프라이즈 수준의 정확도와 오류 감지 기능을 제공합니다.
단순한 카운팅을 넘어, 복잡한 제형 인식과 wrong drug flagging까지 실시간으로 처리합니다. 사람이 놓칠 수 있는 오류를 시스템이 먼저 잡아내어, 약사가 최종 판단에만 집중할 수 있는 환경을 만듭니다.

2. Real-Time Workflow Automation
설치도, 적응할 시간도, 기다림도 필요하지 않습니다.

Instant deployment with seamless workflow integration.

즉시 배포 가능하며 기존 워크플로우에 자연스럽게 통합됩니다.
인력 부족 상황에서도 즉시 투입 가능한 Zero-install 구조로, 기존 시스템을 바꾸지 않고 그대로 연결됩니다. 현장이 기술에 맞춰야 하는 부담 없이, 도입 첫날부터 운영 효율이 올라갑니다.

3. Scalable Cost Efficiency
초기 투자 비용 없이, 바로 사용할 수 있습니다.

Subscription-based model designed for enterprise rollout.

엔터프라이즈 확장을 고려한 구독 기반 모델을 제공합니다.
하드웨어 구매 없이 구독 방식으로 시작하여 CapEx를 제거하고, 체인 약국·병원 그룹 단위의 대규모 확장도 동일한 구조로 대응합니다. 규모가 커질수록 효율도 함께 커집니다.

Foundation / Type

Text

일관된 타이포그래피 시스템을 제공하여 다양한 UI 구성 요소에 적용할 수 있도록 합니다.

Display

화면에서 가장 크고 강조되어야 하는 요소에 사용합니다. 주로 카운팅 숫자, 핵심 수치처럼 한눈에 들어와야 하는 단일 정보에 적합합니다. 숫자로 쓰일 때는 3% 자간을, 텍스트로 쓰일 때는 0% 자간을 적용합니다.

Heading

화면 내 섹션을 구분하거나 팝업/모달의 제목에 사용합니다. Display보다 작지만 Body보다 크고 굵어서 위계를 구분해줍니다.

Body

UI에서 가장 많이 쓰이는 텍스트 유형입니다. 버튼 레이블, 설명 텍스트, 상태 메시지처럼 읽어야 하는 텍스트에 사용합니다.

Caption

화면에서 가장 작은 텍스트입니다. 주요 정보를 보조하는 역할입니다. 단독으로 쓰기보다 다른 텍스트와 함께 사용합니다.

Don't

  • 한 화면에 display는 1~2개 이내로 제한합니다. 많을수록 강조 효과가 희석됩니다.
  • Display는 긴 문장에 사용하지 않습니다. (20글자 이하로 사용)
  • 숫자 단독 표시 시 tracked 변형(자간 3%)을 사용해요. 텍스트 혼용 시엔 기본(0%)을 사용합니다.
  • Caption은 항상 보조 역할입니다. 사용자가 반드시 읽어야 하는 정보에는 쓰지 않아요. Ex) Rx 번호나 NDC 같은 핵심 식별 정보는 body 계열로 올리고, Caption은 부가 정보에만 사용
  • 버튼 레이블에 Caption을 사용하지 않습니다. Caption(14px)은 터치 영역에서 가독성이 떨어지기 때문에 버튼은 최소 Body-3(18px) 이상을 사용해야 합니다.
Foundation / Shape

Radius

XL~XS 상대 크기 체계로 반지름 값을 변환한 토큰입니다. 시각적 중요도·형태 비율·레이아웃 맥락에 맞춰 일관된 Radius를 적용합니다.

Don't

  • 같은 위계의 버튼에 다른 Radius를 쓰지 않습니다. Ex) Cancel Fill, Partial Fill, Complete Fill은 같은 위계의 버튼이므로 동일한 Radius 값을 사용합니다.
  • 작은 컴포넌트에 큰 Radius를 쓰지 않습니다. Ex) 작은 칩/태그 컴포넌트는 corner-radius-XS(4px) 또는 corner-radius-S(6px)를 사용하고 circle은 완전한 원형이 필요할 때만 사용합니다.
  • 모달/다이얼로그에 작은 Radius를 쓰지 않으며 corner-radius-L(12px) 이상부터 적용해서 사용합니다.
  • 중첩된 박스에 같은 Radius를 쓰지 않습니다. 안쪽 박스는 바깥 박스보다 한 단계 작은 Radius를 사용합니다. Ex) 바깥 XL(16px) → 안쪽 M(8px) 또는 S(6px).
  • Radius를 0으로 쓰지 않습니다. 최소 corner-radius-XS(4px) 이상을 사용해요. Radius 0은 토큰에 정의되지 않은 값입니다.
  • 토큰에 없는 임의의 Radius 값을 쓰지 않습니다. 반드시 정의된 토큰 중에서 선택합니다.
